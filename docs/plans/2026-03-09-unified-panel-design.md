### Project Info Card

- **Summary**: Auto-generated by user's agent lightweight model
  - Trigger: auto on returning to е°ҲжЎҲ tab
  - Throttle: min interval (e.g. 5 min) + only regenerate if sessions have new progress
  - Cache last result, show cached + spinner during reload
  - Refresh button: force regenerate
- **Next Steps**: Extracted from each session's ProgressReport.nextSteps, labeled with source session/agent
- **Voice button**: Project-level voice input (full overlay flow)
- **Visual**: Full width, distinct background/border from session cards

### Session Card

- **Display**: Status dot (blue/green/red) + agent name + latest progress summary
- **No session ID displayed**
- **Voice button**: Session-level voice input (full overlay flow)
- **Expand button [v]**: Expand in-place to show metadata (start time, message count, file changes, labels)
- **Tap**: Enter fullscreen event/terminal/diff view (existing behavior)
- **Long press**: Existing menu (rename, delete, label management)

### Project Buttons

- **[...]**: Merge to main, health report, close project, view PRD, edit, delete
- **[вҸ°]**: Project automation/scheduling
- **[+]**: New session (existing NewSessionSheet)

### Tab Navigation

- Top tab bar: tap to switch
- Swipe gesture: left/right to switch between tabs
- Spring animation consistent with current swipe behavior

## Unchanged

- All existing features preserved
- InputBar, full voice flow, NewSessionSheet, AutomationSheet logic
- Event/terminal/diff views
- Glass morphism design, dark/light separate palettes
- Long press gestures, i18n
- CLI/backend untouched

## Summary Generation

- API endpoint: `/api/project-summary`
- Input: all session events/progress for a project
- Model: user's agent lightweight model (e.g. claude-haiku)
- Response: short summary string (1-3 sentences)
- Client caches result + timestamp per project
- Auto-trigger on tab focus, skip if < 5 min since last + no new progress
- Refresh button bypasses throttle
